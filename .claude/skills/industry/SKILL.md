# Skill: industry

산업 분석 스킬 - 심층 산업 컨텍스트 분석

## 목적
클라이언트가 속한 산업의 구조, 트렌드, 동인을 심층 분석합니다.

## 트리거
- `/industry <client-name>` 명령어 사용 시
- 사용자가 "산업 분석", "시장 분석" 요청 시

## 입력
- `clients/<client>/brief.md`
- `clients/<client>/sources.md`

## 출력
- `clients/<client>/research/industry_analysis.md`
- `clients/<client>/sources.md` 업데이트 (#market 섹션)

## 분석 프레임워크

### 1. 시장 구조 분석
- 시장 규모 (TAM/SAM/SOM)
- 성장률 및 전망
- 세분화 (지역, 제품, 고객)
- 가치 사슬

### 2. 산업 동인 분석
- 성장 동인 (Growth Drivers)
- 억제 요인 (Restraints)
- 기회 (Opportunities)
- 위협 (Threats)

### 3. 경쟁 환경 (Porter's 5 Forces)
- 기존 경쟁자 간 경쟁 강도
- 신규 진입자 위협
- 대체재 위협
- 공급자 교섭력
- 구매자 교섭력

### 4. 규제 및 정책 환경
- 주요 규제
- 정책 방향
- 컴플라이언스 요구사항

### 5. 기술 트렌드
- 핵심 기술 변화
- 디지털 전환 동향
- 혁신 사례

## 절차

### Step 1: 산업 정의
클라이언트의 주요 사업 영역과 산업 분류를 명확히 합니다.

### Step 2: 2차 자료 수집
- 산업 보고서 (Gartner, IDC, McKinsey 등)
- 정부/협회 통계
- 뉴스 및 분석 기사

### Step 3: 프레임워크 적용
위의 분석 프레임워크를 순차적으로 적용합니다.

### Step 4: 시사점 도출
클라이언트 관점에서의 전략적 시사점을 도출합니다:
- "So What?" 질문에 답하기
- 기회와 위협 우선순위화
- 액션 가능한 인사이트

### Step 5: 문서화
분석 결과를 `industry_analysis.md`에 구조화하여 기록합니다.

## 출력 템플릿

```markdown

## 3. 경쟁 환경
- 주요 플레이어: ...
- 경쟁 강도: 높음/중간/낮음

## 4. 규제 환경
- 주요 규제: ...
- 정책 방향: ...

## 5. 클라이언트 시사점
- 기회: ...
- 위협: ...
- 권고 액션: ...

## 출처
- sources.md#market 참조
```
