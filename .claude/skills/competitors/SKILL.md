# Skill: competitors

경쟁사 분석 스킬 - 경쟁 환경 심층 분석

## 목적
클라이언트의 주요 경쟁사를 분석하고 전략적 시사점을 도출합니다.

## 트리거
- `/competitors <client-name>` 명령어 사용 시
- 사용자가 "경쟁사 분석", "벤치마킹" 요청 시

## 입력
- `clients/<client>/brief.md`
- `clients/<client>/sources.md`
- `clients/<client>/research/industry_analysis.md` (있는 경우)

## 출력
- `clients/<client>/research/competitor_landscape.md`
- `clients/<client>/sources.md` 업데이트 (#competitors 섹션)

## 분석 프레임워크

### 1. 경쟁사 식별
- 직접 경쟁사 (같은 제품/서비스)
- 간접 경쟁사 (대체재)
- 잠재적 진입자

### 2. 경쟁사별 프로필
각 경쟁사에 대해:
- 회사 개요 (규모, 매출, 직원 수)
- 사업 포트폴리오
- 시장 포지셔닝
- 강점/약점
- 최근 동향 (M&A, 신제품, 전략 변화)

### 3. 비교 분석
- 시장 점유율
- 제품/서비스 비교
- 가격 포지셔닝
- 기술 역량
- 고객 기반

### 4. 벤치마킹
- Best Practice 식별
- 차별화 포인트
- Gap 분석

## 절차

### Step 1: 경쟁사 목록 작성
brief.md를 참고하여 분석 대상 경쟁사 3-5개를 선정합니다.

### Step 2: 정보 수집
각 경쟁사별로:
- 공식 웹사이트, IR 자료
- 뉴스 기사
- 산업 보고서
- 제품/서비스 정보

### Step 3: 프로필 작성
경쟁사별 프로필을 작성합니다.

### Step 4: 비교 매트릭스 작성
주요 지표로 비교 매트릭스를 작성합니다:

| 항목 | 클라이언트 | 경쟁사 A | 경쟁사 B | 경쟁사 C |
|------|-----------|---------|---------|---------|
| 매출 | $XB | $XB | $XB | $XB |
| 시장점유율 | X% | X% | X% | X% |
| 강점 | ... | ... | ... | ... |
| 약점 | ... | ... | ... | ... |

### Step 5: 시사점 도출
- 클라이언트의 경쟁 우위/열위
- 벤치마킹 포인트
- 전략적 권고
