# Consulting Quality Benchmark

## 목적
엔진 산출물을 전략 컨설팅 펌(맥킨지/BCG) 품질 기준에 맞춰 점검하기 위한 내부 벤치마크입니다.

## 평가 프레임

### 1) 스토리라인 구조
- 결론 → 근거 → 실행의 피라미드 구조
- 슬라이드 간 논리 연결(문제정의 → 시사점 → 옵션 → 권고안)
- 중복 메시지 최소화
- 자동 검사: cover 시작, 핵심 타입 존재, 아크 순서 점검

### 2) 메시지 품질
- 거버닝 메시지는 `팩트A + 팩트B = So What` 형식
- claim은 `진단/해석/실행` 3단 구조 권장
- claim마다 `So What:` 포함

### 3) 근거 품질
- 수치 claim은 최소 2개 evidence 교차 검증
- claim ↔ evidence ↔ source 매핑
- 단위/기간/source footer 누락 시 감점

### 4) 레이아웃/디자인 품질
- visual kind 기반 적응형 레이아웃
- slide별 layout decision 로그로 추적 가능
- 타이포 일관성(Calibri), 시각요소 밀도, 정보계층 명확성

### 5) 실행 가능성
- 권고안에 단계(단기/중기/장기)와 KPI를 명시
- 리스크 대응을 실행 로드맵과 연결

## 자동화 보장 수준

자동 보장:
- 스토리 아크 기본 구조
- 근거 매핑 및 출처 추적
- 동적 레이아웃 결정
- QA 점수화 및 fail gate

부분 자동화:
- 문장 과밀 완화
- layout planner의 LLM fallback

사람 보완 필요:
- 산업 특화 인사이트의 현실성
- 조직 의사결정 구조 반영
- 경영진 톤 최종 polish

## 운영 권장

1. 운영 threshold를 85 이상으로 상향하여 내부 품질 기준 강화
2. 제출 전 `qa.summary.md`의 high/medium 이슈를 우선 제거
3. 최종본은 도메인 리더의 1회 리뷰를 의무화
